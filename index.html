MUN-FRL Preprocessing Fixes

The preprocessing script was hardcoded for UZH-FPV racing topics. MUN-FRL uses completely different ROS topic names and formats.

Feature Before (UZH-FPV) After (MUN-FRL)
Image Topic /cam0/image_raw /camera_forward/image_raw
IMU Topic /imu /xsens/imu/data
Action Source /control/command Derived from PPK/GNSS
Image Format BGR8 Mono8 (+ conversion)

Training

NaN Explosion in Stage 2

The Problem

  • Stage 2 unfreezes the Fusion Decoder — vision + IMU cross-attention
  • Learning rate 1×10-3 caused exploding gradients in cross-attention matrix multiplications
  • fp16 precision overflows at much smaller values than fp32 — numbers hit infinity, then NaN
  • Training would crash ~10 epochs into Stage 2, every time

The Fix

  • Reduced learning rates:
    # Stage 2 LR
    lr: 1.0e-3
    lr: 2.0e-4

    # Stage 3 LR
    lr: 1.0e-3
    lr: 3.0e-4
  • Smaller steps let the fusion decoder align vision & IMU signals without oscillating
